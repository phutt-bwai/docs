# How the Conviction Score Works

Every signal you see is built from **multiple independent data sources**, fused through an adaptive weighting engine. Nothing is a black box — every component is visible and decomposable.

```mermaid

    SM[Smart Money Telemetry<br/>Wallet consensus · Value conviction<br/>Behavioral confidence] --> COMP[Composite Signal<br/>Direction + Confidence]
    TA[Technical Analysis<br/>Multi-timeframe indicators<br/>Trend + Momentum + Volatility] --> COMP
    MKT[Derivatives Flow<br/>Funding · OI · Liquidation<br/>Taker flow · L/S ratio] --> COMP
```

**Smart Money Telemetry** is the primary signal source — what fraction of high-credibility wallets are long vs short, how much USD conviction is behind each direction, and how reliable the signal is based on the quality of contributing wallets.

**Technical Analysis** provides timing confirmation across multiple timeframes — when trend structure, momentum, and volatility indicators all align with SM consensus, conviction peaks.

**Derivatives Flow** is the contrarian and confirmation layer — funding extremes, OI surges, liquidation clustering, and taker flow imbalance. When the crowd is positioned one way and SM is positioned the other, something has to give.

The weighting between these sources is **not fixed**. It adapts per asset type and can be tuned per trading agent — a scalping agent weights short-timeframe TA heavily; a whale-following agent prioritizes SM consensus above all else.
